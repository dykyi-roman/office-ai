# src/ — TypeScript Frontend (Svelte 5 + PixiJS v8)

Frontend for the OfficeAI desktop application. Renders a 2D isometric office scene where each running AI agent appears as an animated employee. Built with Svelte 5 (runes), PixiJS v8, and Tauri IPC.

### Root Files

**`main.ts`** — Application entry point. Imports the root `App.svelte` component and mounts it to the `#app` DOM element using Svelte 5's `mount()` API. This is the file referenced by Vite as the entry module.

**`App.svelte`** — Root Svelte component that orchestrates the entire UI. On mount, initializes the settings store, agents store, and sound bridge asynchronously. Creates the `OfficeScene` PixiJS renderer and attaches it to the `#office-canvas` div. Renders 4 overlay components above the canvas: `AgentMetrics` (top-right HUD), settings button (top-left), `AgentSidebar` (slide-in from right), `StatusBar` (fixed bottom), and `SettingsPanel` (modal). Handles `Ctrl+,` keyboard shortcut for settings and `office:select-agent` custom DOM events from the renderer.

**`vite-env.d.ts`** — TypeScript declaration file that adds Vite and Svelte type references. Enables proper IDE support for `import.meta.env`, `.svelte` file imports, and Vite client types.

### `assets/` — Static Assets

**`assets/sprites/agents/`** — Agent spritesheets (4 tiers). Each tier has a `.png` spritesheet and a `.json` atlas file (generated by `scripts/generate-sprites.ts`). Files: `junior.{png,json}`, `middle.{png,json}`, `senior.{png,json}`, `teamlead.{png,json}`. Contains walk, idle, working, and action animation frames.

**`assets/sprites/effects/`** — Visual effect assets. `speech_bubble.svg` and `thinking_indicator.svg` for agent status overlays. Subdirectories: `badges/` (tier badge SVGs: gold, blue, green) and `status_icons/` (tool-use icons: terminal, file, browser, gear, checkmark, exclamation).

**`assets/sprites/furniture/`** — Placeholder for office furniture sprites (desks, chairs, plants). Currently empty (`.gitkeep`).

**`assets/sounds/`** — Placeholder for sound assets. Currently empty — sound is synthesized via Web Audio API in the `sound/` module.

**`assets/themes/`** — Placeholder for theme-specific asset overrides. Currently empty (`.gitkeep`).

**`assets/tiles/`** — Isometric tileset for the office floor. `office-tileset.png` contains tile images, `office-tileset.json` is the atlas definition. Generated by `scripts/generate-tiles.ts`.

### `lib/types/` — TypeScript Type Definitions (Source of Truth)

**`agent.ts`** — Core agent types that Rust structs must mirror exactly. Defines: `Tier` (4 variants: flagship/senior/middle/junior), `Status` (9 variants: idle, walking_to_desk, thinking, responding, tool_use, collaboration, task_complete, error, offline), `IdleLocation` (9 zones), `Source` (cli/browser_extension/sdk_hook), `SubAgentInfo` (id + description), and `AgentState` (14-field interface with id, pid, name, model, tier, status, tokens, sub-agents, etc.).

**`office.ts`** — Office layout types mirrored in Rust. Defines: `LayoutSize` (small/medium/large/campus), `GridPosition` (col, row), `ScreenPosition` (x, y), `DeskAssignment`, `ZoneType` (9 zone types), `Zone` (with capacity and occupants), and `OfficeLayout` (size, dimensions, desks, zones, walkable grid).

**`events.ts`** — Tauri IPC event and command type definitions. Exports `TAURI_EVENTS` constants (agent:found, agent:lost, agent:state-changed, office:layout-changed), typed payload interfaces (`AgentFoundPayload`, `AgentLostPayload`, etc.), `TauriEventMap` for type-safe event listeners, `TAURI_COMMANDS` constants (get_all_agents, get_agent, set_config, get_stats, generate_bug_report), and `AppStats` interface.

**`index.ts`** — Barrel export re-exporting all types from `agent.ts`, `office.ts`, and `events.ts`.

### `lib/stores/` — Svelte 5 Reactive Stores

**`agents.svelte.ts`** — Reactive agent state store using Svelte 5 `$state` and `$derived` runes. Subscribes to Tauri IPC events (`agent:found`, `agent:lost`, `agent:state-changed`). Maintains a `Map<string, AgentState>` with derived values: `activeAgents`, `idleAgents`, `agentsByTier`, `totalTokensIn`, `totalTokensOut`, `totalSubAgents`, `allSubAgents`. Enforces `maxAgents` limit in `addAgent()`. Falls back to mock data (4 sample agents) when Tauri is unavailable (browser dev mode).

**`office.svelte.ts`** — Office state store managing selected agent, sidebar visibility, camera position, and layout size. `selectedAgentId` drives the sidebar (derived `sidebarOpen`). `currentLayoutSize` is derived from agent count: ≤4 = small, ≤10 = medium, ≤20 = large, >20 = campus.

**`settings.svelte.ts`** — User settings store with 10 fields (theme, soundEnabled, showAgentMetrics, animationSpeed, showPrompts, debugMode, scanInterval, maxAgents, customLogPaths, language). Loads from Rust config on init, persists changes via `set_config` IPC command with 400ms debounce. Frontend-only fields (`showPrompts`, `debugMode`, `language`) are not stored in Rust `AppConfig`.

**`index.svelte.ts`** — Barrel export re-exporting all three stores.

### `lib/renderer/` — PixiJS Isometric Scene

**`OfficeScene.ts`** — Main scene orchestrator (~600 lines). Initializes the PixiJS `Application`, loads layouts and spritesheets, manages 5 z-ordered layers (floor → furniture → agents → effects → ui). Subscribes to Tauri events to create/update/remove `AgentSprite` instances. Coordinates `DeskManager`, `IdleZoneManager`, `Pathfinder`, `CameraController`, and `NpcManager`. Handles agent location state machine: idle_zone ↔ walking_to_desk ↔ desk ↔ walking_to_idle.

**`AgentSprite.ts`** — Animated office-employee character extending `PIXI.Container`. Orchestrates body (AnimatedSprite), name label (Text), status indicator (thinking dots, tool icons), and speech bubble. Handles walking animation along grid paths at 120px/s. Tier-specific visual markers (dot colors: gold/blue/green/grey). Coordinates with `AnimationController` for state-driven animations.

**`AnimationController.ts`** — 9-state animation FSM mapping `Status` → animation name. Defines the `AnimatableSprite` interface (decoupled from PixiJS for testability). Manages enter/exit actions per state (show/hide thinking indicator, tool icons, speech bubbles), auto-transitions (e.g., task_complete → idle after animation), and valid transition matrix.

**`Pathfinder.ts`** — A* pathfinding on the isometric boolean grid with 8-directional movement. Features: octile distance heuristic, max 50-tile path length, LRU cache (64 entries) for frequently used routes. Accepts `IdleLocation` → `GridPosition` resolution for zone navigation.

**`DeskManager.ts`** — Desk assignment manager with tier-based priority. Flagship/Senior agents get priority desks; Middle/Junior get standard desks. Priority desks overflow to standard when all standard desks are full. Maintains agent-ID → desk-index mapping for O(1) lookups.

**`IdleZoneManager.ts`** — Assigns idle agents to rest zones (water cooler, kitchen, sofa, etc.) with capacity limits. Rotates agents between zones every 10-30 seconds. Each zone type has an associated animation (drinking, coffee, phone, idle_stand).

**`CameraController.ts`** — Camera pan/zoom/follow controller operating on the PixiJS world container. Supports mouse drag panning, scroll wheel zoom (0.5x-2.0x range with smooth interpolation), keyboard arrows (300px/s), and follow-agent mode with smooth tracking.

**`NpcManager.ts`** — Spawns static NPC characters that roam between zones for office ambience. NPCs are not tied to real processes. Each NPC has a home position, tier, and scheduled routines that drive periodic movement between zone types.

**`SpeechBubble.ts`** — Word-wrapped speech bubble drawn with PixiJS Graphics. Rounded rectangle with tail, scale 0→1 appear animation (150ms), fade-out over 300ms. Auto-dismiss after configurable duration. Max width 200px with text wrapping.

**`utils/isometric.ts`** — Isometric coordinate conversion utilities. Tile dimensions: 128x64 pixels (2:1 ratio). Exports `isoToScreen(col, row)` (grid → pixel), `screenToIso(x, y)` (pixel → grid), `isoDepth(col, row)` (z-sort key), and tile size constants.

**`layouts/medium.json`** — Office layout definition for the medium office size. JSON structure with tile grid, furniture placement, walkable grid (boolean[][]), desk positions (priority + standard), and idle zone definitions (9 zone types with positions and capacities).

### `lib/i18n/` — Internationalization

**`translations.ts`** — Translation dictionaries for 11 supported locales: ar, de, en, es, fr, hi, it, ja, pt, ru, zh. Defines `Locale` type union and `TranslationKey` type. Each locale has a complete dictionary covering app labels, settings, status bar, sidebar, metrics, and all UI strings.

**`index.ts`** — Reactive `t(key)` function that reads the current locale from the settings store and returns the translated string. Falls back to English for missing keys. Exports `SUPPORTED_LOCALES` array with value/label pairs for the language picker.

### `lib/sound/` — Sound System

**`SoundDefinitions.ts`** — Pure data definitions for synthesized sounds. Defines `SoundEvent` type (agent_found, agent_lost, thinking, responding, tool_use, task_complete, error), `OscillatorNote` parameters (frequency, duration, waveform, gain), `SoundDefinition` (notes array + master gain), and `STATUS_SOUND_MAP` mapping agent statuses to sound events.

**`SoundEngine.ts`** — Web Audio API synthesis engine. Lazily creates an `AudioContext` on first use (requires user gesture). Per-event debounce (150ms) prevents rapid-fire sounds. Plays multi-note sequences with attack/release envelopes. Supports volume control and injectable `AudioContextFactory` for testing.

**`SoundEventBridge.ts`** — Bridge between Tauri IPC events and the sound engine. Subscribes independently to `agent:found`, `agent:lost`, and `agent:state-changed` events. Tracks previous agent statuses to only trigger sounds on actual status changes. Activates the AudioContext on first user gesture (click or keydown).

**`index.ts`** — Barrel export re-exporting `SoundEngine`, `initSoundBridge`, `destroySoundBridge`, and all sound definitions/types.

### `lib/ui/` — Svelte Overlay Components

**`AgentMetrics.svelte`** — Floating HUD widget (top-right) showing 4 metric rows: IN (tokens in), OUT (tokens out), AGN (active agent count), SUB (sub-agents). Switches between global and per-agent metrics when an agent is selected in the sidebar.

**`AgentSidebar.svelte`** — Slide-in panel (from right) showing detailed information about the selected agent: name, model, tier badge, status, current task, token counts, sub-agent list, and last activity timestamp. Appears when an agent sprite is clicked.

**`StatusBar.svelte`** — Fixed bottom bar aggregating agents by tier. Shows tier badges with agent counts (e.g., "Flagship: 2, Senior: 3"). Clicking a tier badge filters the view.

**`SettingsPanel.svelte`** — Modal settings panel with 3 sections: General (theme, language, sound), Discovery (scan interval, max agents, custom log paths), Display (animation speed, show metrics). Opened via the settings button or `Ctrl+,`.

**`MiniLog.svelte`** — Compact log viewer for debugging. Shows recent agent events and state transitions in a scrollable list.

**`utils.ts`** — UI utility functions: `formatTokens(n)` (compact number formatting: 1234 → "1.2K"), `formatUptime(seconds)` (human-readable duration), `statusColor(status)` (CSS color per status), `tierColor(tier)` (CSS color per tier), `statusLabel(status)` (localized status names).

**`styles.css`** — Global CSS styles and CSS custom properties (z-index layers, colors, spacing). Defines the visual foundation for all overlay components.

**`index.ts`** — Barrel export re-exporting all 5 Svelte components and utility functions.

## Architecture

### Svelte Stores

#### agents.svelte.ts — Reactive Agent Store

**State** (`$state`):
- `agents: Map<string, AgentState>` — all known agents

**Derived** (`$derived`):
- `activeAgents` — status ≠ idle and ≠ offline
- `idleAgents` — status = idle
- `agentsByTier` — grouped by Tier
- `totalTokensIn`, `totalTokensOut` — sum across all agents
- `totalSubAgents` — count of all active sub-agents
- `allSubAgents` — flat list of all sub-agent infos

**Initialization:** Tauri `listen()` → subscribes to `agent:found/lost/state-changed`. When Tauri is unavailable → mock data (4 sample agents for browser dev mode).

#### settings.svelte.ts — User Settings

10 fields: theme, soundEnabled, showAgentMetrics, animationSpeed, showPrompts, debugMode, scanInterval, maxAgents, customLogPaths, language.

Persistence debounce: 400ms after change → `set_config` via IPC. Frontend-only fields (`showPrompts`, `debugMode`, `language`) persist via `localStorage["officeai-settings"]`, not sent to Rust.

#### office.svelte.ts — Office State

`selectedAgentId`, `cameraPosition`, current layout size (small/medium/large/campus by agent count). `sidebarOpen` is derived from `selectedAgentId !== null`.

Layout size thresholds:
- ≤4 agents → small
- ≤10 → medium
- ≤20 → large
- \>20 → campus

### PixiJS Renderer — 5 Z-Layers

```
Layer 5 (top) │ uiLayer       — UI overlays
Layer 4       │ effectLayer   — visual effects
Layer 3       │ agentLayer    — AgentSprite (z-index = col + row)
Layer 2       │ furnitureLayer — desks, sofas, plants
Layer 1 (bot) │ floorLayer    — floor tiles, walls
```

### OfficeScene — Main Orchestrator

Manages the PixiJS `Application`, rendering, agent sprites, and Tauri event subscriptions. Coordinates `DeskManager`, `IdleZoneManager`, `Pathfinder`, `CameraController`, and `NpcManager`.

**Key constants:**
- `SLOT_OFFSETS`: `[{0,0}, {24,-12}, {-24,-12}, {0,-24}]` — offsets to prevent overlapping
- `DESK_SEAT_OFFSET`: `{32, -16}` px — agent position between desk and chair
- `WORK_STATUSES`: `["thinking", "responding", "tool_use"]`
- `IDLE_STATUSES`: `["idle", "collaboration"]`
- `PRIORITY_DESK_COUNT_MEDIUM`: 2 — lead desks for flagship/senior

**Agent lifecycle:**

```
Appearance (office entrance)
    │
    ├── [showPrompts] Speech bubble: greeting (2s)
    │   └── Localized: "Hello!" / "Привет!" / "Hola!" etc.
    │
    ▼
Walks to idle zone (slotIndex offset)
    │
    ├── Task arrived (status ∈ WORK_STATUSES)
    │   ├── [showPrompts] Speech bubble: currentTask
    │   └── Walks to desk (DESK_SEAT_OFFSET) → work animations
    │       │
    │       ├── Task completed → celebrating → idle zone
    │       └── Error → error icon (5s) → idle zone
    │
    ├── Idle zone rotation (on timer)
    │   └── Move to another zone
    │
    └── Agent disconnected
        ├── [showPrompts] Speech bubble: farewell (2s)
        │   └── Localized: "Bye!" / "Пока!" / "¡Adiós!" etc.
        └── Walks to exit → fade out (1s) → removal
```

**Speech bubbles (controlled by `showPrompts`):**
- **Greeting** — on `agent:found`, before starting movement to idle zone. Duration 2s. i18n key: `agent.greeting`
- **Farewell** — on `agent:lost`, before starting movement to exit. Duration 2s. i18n key: `agent.farewell`
- **Current task** — on transition to work status (thinking/responding/tool_use), shows `agent.currentTask` (up to 200 chars, XML commands filtered)

### A* Pathfinder

8-directional A* pathfinding on the isometric boolean grid.

**Constants:**
- `MAX_PATH_LENGTH`: 50 tiles
- `CACHE_CAPACITY`: 64 entries (LRU)
- 8 movement directions (4 cardinal + 4 diagonal)

**Octile heuristic:**
```
h(a, b) = dx + dy + (√2 - 2) × min(dx, dy)
where dx = |a.col - b.col|, dy = |a.row - b.row|
```

**Diagonal movement restriction:**
To prevent "cutting through corners" — both cardinal cells adjacent to the diagonal must be walkable:
```
if (dc ≠ 0 && dr ≠ 0):
  if !walkable(current.col + dc, current.row) → skip
  if !walkable(current.col, current.row + dr) → skip
```

**LRU cache:** key = `"col,row->col,row"`, on `get()` entry moves to end (MRU), on overflow the first entry (LRU) is evicted.

### DeskManager — Tier-Based Desk Assignment

```
DeskRecord { position, tier: "priority"|"standard", agentId }

Algorithm assignDesk(agentId, tier):
  1. flagship/senior → priority desks first, fallback to standard
  2. middle/junior → standard desks first, fallback to priority
  3. Random selection among free desks of preferred type
  4. If none available → throw Error
```

Default: first 2 desks are priority (for lead positions).

### IdleZoneManager — Idle Zone Management

**9 zone types:** water_cooler, kitchen, sofa, meeting_room, standing_desk, desk, bathroom, hr_zone, lounge.

**Animations by zone type:**

| Zone           | Animation   |
|----------------|-------------|
| water_cooler   | drinking    |
| kitchen        | coffee      |
| sofa, lounge   | phone       |
| others         | idle_stand  |

**Zone dwell time:**
- MIN_IDLE_MS: 10,000 ms
- MAX_IDLE_MS: 30,000 ms
- Randomized: `MIN + random() × (MAX - MIN)`

**Rotation:** when dwell timer expires, agent moves to another zone (preferably not the previous one), respecting capacity limits.

### AnimationController FSM — 9-State Animation FSM

**AnimatableSprite interface** (decoupled from PixiJS for testability):
```typescript
interface AnimatableSprite {
  alpha: number;
  playAnimation(name: string): void;
  showThinkingIndicator(): void;
  hideThinkingIndicator(): void;
  showToolIcon(tool: "terminal"|"file"|"browser"): void;
  hideToolIcon(): void;
  showSpeechBubble(text: string, durationMs?: number): void;
  hideSpeechBubble(): void;
  showTaskComplete(): void;
  showError(): void;
}
```

**State table:**

| Status          | Animation   | Auto-transition           |
|-----------------|-------------|---------------------------|
| idle            | idle_stand  | —                         |
| walking_to_desk | walk_down   | —                         |
| thinking        | thinking    | —                         |
| responding      | typing      | —                         |
| tool_use        | typing      | —                         |
| collaboration   | idle_stand  | —                         |
| task_complete   | celebrating | → idle after 3s           |
| error           | frustrated  | → idle after 5s           |
| offline         | idle_stand  | — (alpha = 0)             |

**Walking queue:** during walking, non-priority transitions (except error/offline) are queued in `pendingStatus` and applied after `walkComplete()`.

### AgentSprite — Visual Agent

**Visual hierarchy (bottom to top):**
1. selectionHighlight — blue ellipse (0x00aaff)
2. body — AnimatedSprite from spritesheet
3. nameLabel — text below feet
4. statusIndicator — indicators above head
5. speechBubble — text bubble

**Walking speed:** 120 px/sec

**Status indicators:**
- **Thinking:** 3 colored dots with sine-bounce animation (stagger 0.8 phase)
- **Tool Use:** colored square with icon (terminal → ">", file → "F", browser → "B")
- **Task Complete:** green checkmark with 2s fade-out
- **Error:** red circle with white cross

**Dot colors by tier:** flagship=gold, senior=blue, middle=green, junior=grey

### CameraController

**Constants:**
- Zoom: 0.5× – 2.0×, step 0.1, smoothing 0.12 per frame
- Pan: 300 px/sec (arrows), middle mouse button
- Follow: smoothing 0.08 per frame

**Controls:**
- Mouse wheel → zoom
- Middle button + drag → pan
- Arrow keys → pan (disables follow)
- +/- → zoom
- `follow(sprite)` → smooth agent tracking

### NpcManager — Background NPCs

Spawns non-player characters that roam between zones on timers for office ambience.

**Routine cycle:**
1. Wait random interval [min, max] ms
2. Find target zone via resolver
3. A* route to zone
4. Dwell in zone [min, max] ms
5. Return to home position
6. Repeat

### SpeechBubble — Animated Bubble

**Phases:** idle → appearing (150ms, scale 0→1, ease-out) → visible → disappearing (300ms, alpha 1→0)

Ease-out: `f(t) = 1 - (1-t)²`

**Rendering:** drop-shadow → white rounded rect → border → triangle tail → centered text.

**Word wrap:** `wordWrap: true`, `breakWords: true` — long words without spaces (URLs, XML tags) are force-broken at bubble boundary (`MAX_BUBBLE_WIDTH = 200px`).

### Isometric Math

**Tile dimensions:** 128×64 px (2:1 ratio, ~30° projection)

**Grid → Screen:**
```
screenX = (col - row) × 64
screenY = (col + row) × 32
```

**Screen → Grid:**
```
col = floor(screenX / 128 + screenY / 64)
row = floor(screenY / 64 - screenX / 128)
```

**Z-order (draw sorting):**
```
zIndex = col + row
```
Higher col+row = "closer" to viewer → drawn on top.

### Sound System — Audio Synthesis

The sound system (`src/lib/sound/`) consists of 3 modules:

**SoundEngine** — Web Audio API synthesis with ADSR envelopes:
- Lazy AudioContext: created only after user gesture (click/keydown) due to browser autoplay policy
- Per-event debounce: 150ms minimum interval between repeats of the same sound
- ADSR: Attack (10ms) → Sustain → Release (50ms)
- Each sound = sequence of `OscillatorNote` (frequency, duration, waveform, gain, detune)

**SoundDefinitions** — 7 synthesized sound effects:

| Event          | Description                                   | Waveform  |
|----------------|-----------------------------------------------|-----------|
| agent_found    | Ascending major chord (C5-E5-G5)              | square    |
| agent_lost     | Descending decay (A4-F4-C4)                   | sawtooth  |
| thinking       | Detuned sinusoidal hum (220Hz +50¢)           | sine      |
| responding     | Quick bright blip (880-1047Hz)                | square    |
| tool_use       | Staccato click (800-600Hz, 40ms)              | square    |
| task_complete  | Victory jingle (G5-B5-D6)                     | square    |
| error          | Dissonant buzz (tritone E4+Bb4)               | sawtooth  |

**SoundEventBridge** — maps Tauri events to sounds:
- Independent subscription to `agent:found`, `agent:lost`, `agent:state-changed`
- Sound plays only when: `soundEnabled == true` + AudioContext activated
- Deduplication: sound plays only on **actual** status change (previous ≠ current)

## Test Structure

Tests are colocated in `__tests__/` directories next to their source:
- `renderer/__tests__/` — 11 test files covering all renderer classes
- `stores/__tests__/` — 4 test files for agents, office, settings, utils
- `sound/__tests__/` — 3 test files for engine, bridge, definitions
- `i18n/__tests__/` — 1 test file for translation lookup
- `ui/__tests__/` — 1 test file for UI utilities

Environment: **Node** (no browser/WebGL). All PixiJS classes must be mocked. The `AnimatableSprite` interface decouples FSM logic from PixiJS for testability.

## Conventions

- **TypeScript strict mode**, no `any`
- **Svelte 5 runes**: `$state`, `$derived`, `$effect` (not legacy stores)
- **camelCase** for variables/functions, **PascalCase** for types/classes
- Import types from `$lib/types/`, never redefine
- Path alias: `$lib` → `./src/lib`
